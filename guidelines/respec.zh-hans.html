使用ReSpec

ReSpec是一个用HTML编写技术规范的工具。你编写一个源HTML文件，添加一个小的配置对象，ReSpec就会把它转换成格式化后的规范，其中包含标题、目录、节编号、参考文献、定义、交叉链接、注记和导出工具。

本指南介绍创建和预览ReSpec文档所需的基础知识。

创建一个最小的ReSpec文件

从一个HTML文件开始，比如index.html：

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <title>My First Document</title>

  <script
    src="https://www.w3.org/Tools/respec/respec-w3c"
    class="remove"
    async></script>

  <script class="remove">
    var respecConfig = {
      specStatus: "ED",
      editors: [
        {
          name: "Your Name",
          url: "https://example.com/"
        }
      ],
      shortName: "my-first-spec",
      github: "w3c/my-first-spec",
      group: "i18n",
      canonicalURI: "TR",
      license: "w3c-software-doc"
    };
  </script>
</head>
<body>
  <section id="abstract">
    <p>This document defines a small example specification.</p>
  </section>

  <section id="sotd"></section>

  <section>
    <h2>Introduction</h2>
    <p>This is where the document begins.</p>
  </section>

  <section id="conformance"></section>
</body>
</html>

这两个script元素很重要。第一个加载ReSpec，第二个告诉ReSpec如何构建文档。

请使用var respecConfig，不要使用const或let。ReSpec会从window.respecConfig读取配置。

class="remove"属性会告诉ReSpec在导出的静态HTML中移除这些编写文件时使用的脚本。

在本地预览文档

不要直接通过file://打开文件，用本地的HTTP服务器打开。

如果你安装了 Python，可以在文件所在目录中运行以下命令：

python3 -m http.server

然后打开：

http://localhost:8000/

通过HTTP访问文档很重要，因为ReSpec可能需要获取参考文献、交叉引用数据、issue信息、包含文件或其他资源。

了解基本配置

一个小型配置通常像这样：

var respecConfig = {
  specStatus: "ED",
  editors: [{ name: "Your Name" }],
  shortName: "my-spec",
  github: "w3c/repo",
  group: "i18n"
};

初学者常用的选项：

  • specStatus：发布状态。编写编辑草案时使用"ED"。
  • editors：文档编辑。大多数W3C文档至少需要一位编辑。
  • shortName：文档的短标识符。
  • github：用于反馈的仓库。
  • group：负责该文档的W3C小组。

编写章节

ReSpec文档就是HTML文档，请使用带标题的section元素。

<section>
  <h2>Concepts</h2>
  <p>A <dfn>widget</dfn> is an object managed by this specification.</p>

  <section class="informative">
    <h3>Background</h3>
    <p>This section gives non-normative background information.</p>
  </section>
</section>

ReSpec会自动为章节编号并生成目录。如果你的文档是规范性文档，请对非规范性的解释性材料使用class="informative"。

如果你使用空链接指向某个章节，ReSpec会自动填入章节编号和标题：

<p>See <a href="#concepts"></a>.</p>

定义与链接术语

使用dfn定义术语：

<p>A <dfn>request</dfn> represents a pending operation.</p>

在后文链接到该术语：

<p>Each [=request=] has a state.</p>

也可以使用普通的HTML链接：

<p>Each <a>request</a> has a state.</p>

如果一个术语有多个名称，请使用data-lt：

<p>A <dfn data-lt="user agent|browser">user agent</dfn> loads the page.</p>

这样，[=user agent=]和[=browser=]都会链接到同一个定义。

链接到其他规范

ReSpec可以链接到其他规范中定义的术语。请在配置中启用交叉引用：

var respecConfig = {
  xref: "web-platform"
};

然后使用ReSpec的简写：

<p>Queue a task to [=fire an event=] at the {{Window}} object.</p>

常见的简写形式：

简写 用途
[=term=] 链接到概念或定义。
{{Interface}} 链接到WebIDL接口。
{{Interface/member}} 链接到WebIDL属性或方法。
[^element^] 链接到HTML或SVG元素。
[[HTML]] 添加资料性引用。
[[!RFC2119]] 添加规范性引用。

如果ReSpec找不到交叉引用，它会在渲染后的文档中显示警告或错误。

添加引用

<p>The [^link^] element is defined in [[HTML]].</p>
<p>Conforming implementations MUST follow [[!RFC2119]].</p>

使用[[SPEC]]表示资料性引用，[[!SPEC]]表示规范性引用。

ReSpec会自动生成参考文献章节。

添加示例、注、issue和图

用note提供有帮助的解释：

<aside class="note" title="Implementation note">
  <p>Implementations should handle network failure.</p>
</aside>

用example提供样例代码或具体场景：

<aside class="example" title="Using the API">
  <pre class="js">
const result = await doSomething();
  </pre>
</aside>

用issue表示尚未解决的编辑性或技术性问题：

<p class="issue">
  Decide whether this algorithm should reject or resolve.
</p>

添加带说明文字的图：

<figure id="fig-overview">
  <img src="overview.svg" alt="System overview diagram">
  <figcaption>System overview.</figcaption>
</figure>

<p>See <a href="#fig-overview"></a>.</p>

ReSpec可以为图编号，并填充指向图的空链接。

需要时使用Markdown

ReSpec可以把整个文档正文当作GitHub风格Markdown来处理：

var respecConfig = {
  format: "markdown"
};

使用此模式时，请让Markdown内容与左边对齐，因为Markdown对空白字符敏感。

也可以在一个章节中包含Markdown文件：

<section data-include="my-section.md"
         data-include-format="markdown"></section>

HTML和Markdown可以混用，但HTML元素内的Markdown内容前后需要留空行。
